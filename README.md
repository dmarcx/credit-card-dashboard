# 💳 דשבורד הוצאות אשראי

ניתוח חכם של פירוטי כרטיס אשראי **כאל** ו-**ישראכרט** — מ-PDF לדשבורד אנליטי בשניות.

![Python](https://img.shields.io/badge/Python-3.10+-blue) ![Streamlit](https://img.shields.io/badge/UI-Streamlit-red) ![License](https://img.shields.io/badge/License-Personal-lightgrey)

---

## מה זה עושה?

מעלים קבצי PDF של פירוטי כרטיס אשראי → האפליקציה מחלצת את כל העסקאות אוטומטית → ומציגה דשבורד אינטראקטיבי:

| פיצ'ר | תיאור |
|---|---|
| **חילוץ PDF** | תמיכה בכאל וישראכרט, כולל תיקון עברית RTL אוטומטי |
| **ריבוי כרטיסים** | העלאת כאל + ישראכרט לאותו חודש — מיזוג אוטומטי לתצוגה אחת |
| **חודשי חיוב** | כל עסקה משויכת לחודש החיוב של הקובץ (לפי שם הקובץ), לא לתאריך העסקה |
| **KPI cards** | סה"כ הוצאות, ממוצע חודשי, מספר עסקאות |
| **גרף עוגה** | התפלגות הוצאות לפי קטגוריה (אינטראקטיבי) |
| **ספקים מובילים** | Top 8 בתרשים עמודות אופקי |
| **סיכום קטגוריות** | טבלה מקובצת עם סך הוצאה לכל קטגוריה |
| **תובנות וחריגים** | זיהוי עסקאות חריגות וקפיצות בהוצאות |
| **טבלת עסקאות** | כל העסקאות עם תאריך ביצוע, חיפוש, מיון וסינון |
| **סינון חודשים וקטגוריות** | סינון דינאמי — כולל החרגת משיכות מזומנים ורכישות חו"ל |
| **מצב בהיר/כהה** | מתג תצוגה בסרגל הצד |

> **פרטיות:** קבצי PDF נשארים אצלך בלבד ולא מועלים לשום מקום.

## פורמטים נתמכים

| כרטיס | פורמט | הערות |
|---|---|---|
| **כאל** (ויזה / מאסטרקארד / דיינרס) | דף פירוט דיגיטלי (PDF במייל) | ניתוח regex ישיר, ללא API |
| **ישראכרט** (מאסטרקארד) | פירוט חודשי (PDF במייל) | ניתוח regex ישיר, ללא API |
| **אחר** | כל PDF של פירוט אשראי | Fallback דרך Claude API (דורש מפתח) |

### שמות קבצים — חשוב

האפליקציה קוראת את **חודש החיוב משם הקובץ**. ודא ששם הקובץ מכיל את החודש והשנה:

```
כאל 01-26.pdf          ✅  ינואר 2026
ישראכרט פברואר 26.pdf  ✅  פברואר 2026
כאל 03-26.pdf          ✅  מרץ 2026
statement.pdf          ⚠️  Fallback — משתמש בחודש הנפוץ ביותר בעסקאות
```

## התקנה

### דרישות מוקדמות

- Python 3.10 ומעלה
- Git

### שלב 1 — שכפול הריפו

```bash
git clone https://github.com/dmarcx/credit-card-dashboard.git
cd credit-card-dashboard
```

### שלב 2 — יצירת סביבה וירטואלית (מומלץ)

```bash
python -m venv .venv

# Windows
.venv\Scripts\activate

# Mac / Linux
source .venv/bin/activate
```

### שלב 3 — התקנת תלויות

```bash
pip install -r requirements.txt
```

### שלב 4 — הרצה

```bash
streamlit run app.py
```

הדפדפן ייפתח אוטומטית בכתובת `http://localhost:8501`.

## שימוש

1. **העלאת PDF** — גרור קבצי פירוט כאל / ישראכרט לאזור ההעלאה בסרגל הצד. ניתן להעלות **מספר חודשים ומספר כרטיסים** בו-זמנית.
2. **סינון חודשים** — בחר חודשי חיוב לצפייה ממוקדת. לחץ **הכל** לתצוגה מלאה.
3. **סינון קטגוריות** — הסתר קטגוריות כגון `משיכת מזומנים` או `רכישות חו"ל` לסיכום נקי.
4. **חיפוש ספק** — בטבלת העסקאות, חפש לפי שם בית עסק.
5. **נתוני הדגמה** — ללא PDF? לחץ על "נתוני הדגמה" לראות את הדשבורד עם נתונים מדומים.

## מבנה הפרויקט

```
credit-card-dashboard/
├── app.py            # ממשק משתמש — Streamlit בלבד, ללא לוגיקה עסקית
├── parser.py         # חילוץ נתונים מ-PDF (כאל, ישראכרט, fallback Claude API)
├── analytics.py      # חישובי KPI, אגרגציות, זיהוי חריגים, נתוני הדגמה
├── requirements.txt  # תלויות Python
└── README.md
```

### זרימת הנתונים

```
PDF קובץ (כאל / ישראכרט)
    ↓
parser.py → parse_pdf()
    ↓ DataFrame: date | merchant | amount | category
app.py → _process_uploads()  ← מוסיף עמודת billing_month לפי שם קובץ
    ↓ DataFrame: date | merchant | amount | category | billing_month
analytics.py → compute_kpis() / get_category_breakdown() / get_anomalies()
    ↓ מדדים מחושבים
app.py → Streamlit renders dashboard
```

הפרויקט מוגדר עם **Stop hook** של Claude Code שמבצע `git commit` ו-`git push` אוטומטית בסוף כל סשן פיתוח. ר' [`.claude/settings.json`](.claude/settings.json).

## פתרון בעיות

**הפירוט לא מזוהה**
- ודא שהקובץ הוא פירוט כרטיס **כאל** או **ישראכרט**.
- הפורמט הנתמך הוא "דף פירוט דיגיטלי" שנשלח במייל.
- לפורמטים אחרים, הגדר `ANTHROPIC_API_KEY` בקובץ `.streamlit/secrets.toml`.

**חודש חיוב שגוי**
- ודא ששם הקובץ מכיל את החודש והשנה (ר' טבלת פורמטים למעלה).

**שמות ספקים הפוכים**
- לא אמור לקרות — `parser.py` מטפל בתיקון RTL אוטומטי.
- אם אתה רואה טקסט הפוך, פתח issue עם דוגמה (ללא מידע אישי).

**שגיאת התקנה**
```bash
pip install --upgrade pip
pip install -r requirements.txt
```
